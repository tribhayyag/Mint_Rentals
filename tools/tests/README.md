# Wizard → tools integration tests

Verifies that the React onboarding wizard's `UserPreferences` output is shape-compatible with the Python tools in `tools/`. There are two suites:

| Suite | What it verifies | Requires |
|-------|-----------------|----------|
| `test_wizard_to_tools.py` | Schema parity, every wizard input path, every tool's filter, fuzz | Python 3 only |
| `test_real_ts.sh` + `test_real_ts.py` | The Python sim in `wizard_sim.py` matches the **actual compiled TypeScript** | Node + `npx tsc` |

## Run everything

From repo root:

```bash
bash tools/tests/run_all.sh
```

Exits 0 on success, 1 on any failure.

## Run individually

```bash
# Python-only suite (fast)
python3 tools/tests/test_wizard_to_tools.py

# Real-TS drift test (compiles toUserPreferences.ts and runs it via Node)
bash tools/tests/test_real_ts.sh
```

## Files

- `runner.py` — minimal test framework (no external deps).
- `wizard_sim.py` — Python mirror of `frontend/src/lib/toUserPreferences.ts`. **Keep in sync with the TS source**; `test_real_ts.py` will detect drift.
- `test_wizard_to_tools.py` — main integration suite.
- `wizard_driver.mjs` — Node script that imports the compiled TS and prints JSON for known inputs.
- `test_real_ts.sh` — compiles the TS module, runs the Node driver, hands off to `test_real_ts.py`.
- `test_real_ts.py` — compares real TS output to `wizard_sim`, plus re-runs pipeline checks against the real output.
- `_compiled/` — generated by `test_real_ts.sh`; safe to delete.

## When to update these tests

| You changed... | Update... |
|----------------|-----------|
| `frontend/src/lib/toUserPreferences.ts` | `wizard_sim.py` (keep them equivalent) |
| `frontend/src/types/preferences.ts` shape | both `wizard_sim.py` and tests that reference the schema |
| A Python tool's input/output envelope | the relevant `pipeline:` tests in `test_wizard_to_tools.py` |
| Added a new wizard step | add a corresponding mapping in `wizard_sim.py` + tests covering the new path |
